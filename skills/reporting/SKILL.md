# Reporting — honest status from the ledger

Thin Cambium-way wrapper. Hand off to the **Orchestrator**; run the reporting phase.

## Run it
1. **Reporting · reporting-officer** — build the report from the findings ledger, run history, and
   verified results: what was done, milestones vs plan, what's next, and risks/blockers. State only
   results that carry a **code-verified** or **proved** tier; flag anything still **asserted** or
   **open** as such. No spin.
2. **Reporting · deck-builder** (only if a deck is asked for) — turn the approved report into slides
   (PPTX/HTML): progress review, annual update, or pitch.

Pull numbers only from the ledger/verified results — never restate an unverified claim as fact.

## Gate
Stop at gate card **G5 — "release report?"**: decision · options · risks · recommendation ·
APPROVE / REVISE / REJECT. Release or send only after the Director approves.
